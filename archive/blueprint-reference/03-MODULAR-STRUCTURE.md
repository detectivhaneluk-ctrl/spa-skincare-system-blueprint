# 03 — Վերջնական մոդուլային կառուցվածքը

## Գլխավոր Կառավարման Մոդուլներ
1. Նույնականացում և անվտանգություն
2. Գլխավոր վահանակ
3. Օրացույց և ամրագրումներ
4. Հաճախորդներ
5. Վաճառք, վճարումներ և դրամարկղ
6. Նվեր քարտեր և փաթեթներ
7. Ապրանքներ և պահեստ
8. Ծառայություններ, սենյակներ և սարքավորումներ
9. Անձնակազմ
10. Հաշվետվություններ և վերլուծություն
11. Մարքեթինգ, լոյալթի և հաղորդակցություն
12. Փաստաթղթեր և համաձայնություններ
13. Կարգավորումներ
14. Առցանց ամրագրում
15. Համակարգային միջուկ

- Հաճախորդներ
- Վաճառք

- Ապրանքներ և պահեստ

- Անձնակազմ

### Անտեսանելի համակարգային շերտ
- նույնականացում
- իրավասություններ
- կարգավիճակների շարժիչ
- հոսքերի շարժիչ
- audit log
- notification queue
- tax engine
- pricing engine
- file engine
- search engine
- conflict engine
- soft delete / archive
- backup / restore

## Չխառնելու կանոն
UI-ում երբեք մի՛ գրեք այնպիսի բիզնես որոշումներ, որոնք պետք է ապրեն միջուկում։
Օրինակ՝
- կարգավիճակի փոփոխման կանոնները
- հարկային հաշվարկը
- փաթեթի օգտագործման տրամաբանությունը
- բախումների կանխումը
- իրավասությունների ստուգումը
